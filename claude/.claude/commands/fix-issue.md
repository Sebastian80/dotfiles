description: Fix a GitHub/Jira issue end-to-end
---

Fix issue **$ARGUMENTS** following this workflow:

## 1. Understand the Issue
- Read the issue details (Jira/GitHub)
- Identify acceptance criteria
- Clarify any ambiguities

## 2. Locate Relevant Code
- Find files related to the issue
- Understand the current implementation
- Identify what needs to change

## 3. Plan the Fix
- Design the solution
- Consider edge cases
- Plan test coverage

## 4. Implement
- Make the necessary changes
- Follow existing code patterns
- Keep changes minimal and focused

## 5. Verify
- Run relevant tests
- Run static analysis (composer analysis)
- Test manually if needed

## 6. Commit
- Create a commit with message referencing the issue
- Format: `HMKG-XXX: Brief description of the fix`
